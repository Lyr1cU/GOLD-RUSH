## 4. Дизайн: чорно-золотистий

### Палітра (`:root` у `css/styles.css`)

```css
:root {
  --color-bg: #0c0c0c;
  --color-bg-alt: #141414;
  --color-surface: #1a1a1a;
  --color-surface-elevated: #222222;
  --color-text: #f5f5f5;
  --color-text-muted: #a3a3a3;
  --color-border: #2e2e2e;
  --color-accent: #d4af37;        /* золото */
  --color-accent-hover: #e8c84a;
  --color-accent-soft: rgba(212, 175, 55, 0.12);
  --color-hero-overlay: rgba(0, 0, 0, 0.55);
}
```

### Стиль

- **Фон:** темний, майже чорний
- **Акценти, кнопки, hover, підкреслення:** золото
- **Картки меню:** темно-сірі з золотою рамкою або акцентною ціною
- **Шрифт:** Inter (вже в шаблоні) — для заголовків можна **700**, великий hero
- **Hero:** темне фото бургерів / фастфуду (плейсхолдер або сток), overlay чорний
- **Відчуття:** преміальний фастфуд, не «дешевий шаблон»

### Адаптив

Перевірити: **375px**, **768px**, **1024px+**.

## 5. Структура сайту

### 5.1 Головна — `index.html`

Секції (порядок):

1. **Header** — лого, навігація (якорі + посилання на сторінки), UA/EN/RU, CTA «Замовити»
2. **Hero** — заголовок, підзаголовок, кнопка «Меню» / «Замовити»
3. **Акції та комбо (превью)** `#promos` ← **нова секція (+1)**  
   - 2–3 картки комбо (напр. «Комбо Класик», «Подвійний бургер»)  
   - кнопка **«Усі акції →»** на `promos.html`
4. **Меню (превью)** `#menu` — 6–8 позицій + кнопка «Повне меню →» на `menu.html`
5. **Про нас** `#about` — 2–3 речення про мережу
6. **Галерея** `#gallery` — 6 фото (інтер'єр, їжа)
7. **Відгуки** `#reviews` — 3 відгуки
8. **Контакти** `#contact` — **перемикач локацій** (див. §5.4), телефон, форма, **карта під обрану адресу**
9. **Footer**

### 5.2 Додаткові сторінки (×3)

| Файл | Зміст |
|---|---|
| **`menu.html`** | Повне меню: **4+ категорії**, **20+ позицій** (бургери, картопля, напої, десерти). Таби або фільтр категорій. Спільний header/footer. |
| **`promos.html`** | Повний каталог **акцій та комбо**: 6–8 карток, ціни, опис, фото, умови (напр. «діє до …»). Той самий стиль, що превью на головній. |
| **`contacts.html`** | Повна сторінка контактів: **перемикач локацій** (як на головній), адреса + графік обраної точки, телефон, соцмережі, форма заявки, **карта**, опційно список усіх адрес текстом. |

Навігація в header на всіх сторінках:

- Головна (`index.html`)
- Меню → `menu.html`
- Акції → `promos.html`
- Контакти → `contacts.html`

### 5.3 Перемикач локацій + карта (головна і contacts.html)

**Задача:** у бізнеса **кілька точок** (2–3). Користувач обирає локацію **табом / чіпом** (назва точки, напр. «Центр», «Набережна») — і **адреса, графік та карта** оновлюються під неї.

**Де працює:**
- секція `#contact` на **`index.html`**
- повністю на **`contacts.html`**

**UI (приклад):**

```
[ Центр ]  [ Набережна ]  [ Караван ]   ← активний чіп золотий

Адреса:    пр. Дмитра Яворницького, 1, Дніпро
Графік:    10:00–23:00 щодня

[        Google Maps iframe         ]
```

**Поведінка:**
1. За замовчуванням активна **перша** локація з JSON.
2. Клік по чіпу → оновити текст адреси/графіку + **перебудувати iframe карти** (lat/lng/zoom).
3. Один і той самий JS-модуль на головній і на `contacts.html` (напр. `js/render/locations.js`).

**Дані — `content/contacts.json`:**

```json
{

  "title_ru": "Контакты",
  "phoneLabel_uk": "Телефон",
  "phone": "+380670000001",
  "phoneDisplay_uk": "+380 67 000 00 01",
  "mapLabel_uk": "Карта",
  "locations": [
    {
      "id": "center",
      "name_uk": "Центр",

      "name_ru": "Центр",
      "address_uk": "пр. Дмитра Яворницького, 1, Дніпро",
      "address_en": "1 Dmytra Yavornytskoho Ave, Dnipro",
      "address_ru": "пр. Дмитра Яворницького, 1, Днепр",
      "hours_uk": "10:00–23:00 щодня",
      "hours_en": "10:00–23:00 daily",
      "hours_ru": "10:00–23:00 ежедневно",

      "mapZoom": 16
    }
  ],
  "socials": [ ... ]

```

**Техніка карти** (як у Zerno — `js/render/contacts.js`):
- Google Maps embed: `https://www.google.com/maps?q={lat},{lng}&hl={uk|en|ru}&z={zoom}&output=embed`
- При зміні локації — оновити `iframe.src`, не створювати новий iframe щоразу (або `replaceChildren` — як у Zerno).

**Адмінка (Decap):**
- Колекція / список `locations` з полями: `id`, `name_*`, `address_*`, `hours_*`, `mapLat`, `mapLng`, `mapZoom`
- Клієнт додає/редагує точки в адмінці → сайт підхоплює

Додати `content/promos.json` — акції/комбо (аналогічно menu: name_*, desc_*, price_*, image).

**Мінімум локацій у демо:** **2** (можна 3).

### 5.4 Мови

| Мова | Код |
|---|---|
| Українська | `uk` (default) |
| English | `en` |
| Російська | `ru` |

Додати `js/i18n/ru.js`, оновити перемикач на 3 кнопки. Усі JSON: поля `_uk`, `_en`, `_ru` (або розширити існуючу схему послідовно).
